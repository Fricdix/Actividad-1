1. Ventajas de la versión orientada a objetos:

La programación orientada a objetos ofrece una organización más clara y lógica del código. Al encapsular los datos y comportamientos en clases, el programa se vuelve más modular y mantenible. Es más fácil reutilizar el código en otras partes del sistema y escalar la aplicación agregando nuevas funcionalidades sin afectar el código existente.
2. Parte del código más clara o fácil de mantener:

La creación de la clase Producto resultó especialmente beneficiosa, ya que centraliza toda la información y operaciones relacionadas con un producto en un único lugar. Métodos como calcularTotal() y mostrarInfo() hacen que el código sea más intuitivo y legible, facilitando las futuras modificaciones y el debugging.
3. Extender el programa para más productos:

Para manejar una cantidad dinámica de productos, se podría implementar una lista en lugar de un array estático. Esto permitiría agregar, eliminar y modificar productos de manera flexible, sin las limitaciones de un tamaño fijo. Además, se podrían incorporar funciones de búsqueda y filtrado para mejorar la gestión del inventario.
4. Agregar una clase Tienda:

Definitivamente, una clase Tienda sería muy útil para centralizar la gestión completa del inventario. Esta clase podría encargarse de almacenar todos los productos, calcular totales generales, gestionar promociones y generar reportes, siguiendo el principio de responsabilidad única y mejorando la organización del código.
